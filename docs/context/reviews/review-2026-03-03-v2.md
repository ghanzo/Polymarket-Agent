## Test Suite Assessment

**Grade: B** (up from B-)

### What Improved
- **Fee accounting coverage**: Excellent — `test_fee_accounting.py` (257 lines) and `test_balance_lifecycle.py` (770 lines) call real production functions and verify SQL parameter values
- **BalanceTracker pattern**: Intercepts actual `resolve_bet()`/`close_bet()` calls via `_FakeCursor`, verifying computed values — much better than inline reimplementation
- **Algebraic invariant testing**: Parametrized `TestAlgebraicInvariant` proves `balance_add = amount + pnl` across multiple scenarios
- **Vacuous assertion removed**: The all-values-accepted assertion is gone

### What's Still Weak
- ~15% of tests still reimplement production logic inline (trailing stops, event dedup, time scoring, some fee math)
- All DB tests still mock — no real PostgreSQL execution
- 5+ `inspect.getsource()` tests remain (brittle)
- test_strategies.py time tests still use `datetime.now()` without freezegun
- 8 different `make_market()` implementations with diverging defaults
- New BalanceTracker SQL parsing is fragile (string-matching against SQL templates)
